    territory_from_locale(locale)

    (since 2.18.2)

    Specs

    territory_from_locale(Cldr.LanguageTag.t() | Cldr.Locale.locale_name()) ::
  Cldr.Locale.territory()

    Returns the territory from a language tag or locale name.

    Arguments

    • locale is any language tag returned by Bonfire.Web.Cldr.Locale.new/1 or a locale name in the list returned by Bonfire.Web.Cldr.known_locale_names/0

    Returns

    • A territory code as an atom

    Examples 

    iex> Bonfire.Web.Cldr.Locale.territory_from_locale "en-US"
:US

iex> Bonfire.Web.Cldr.Locale.territory_from_locale "en-US-u-rg-GBzzzz"
:GB

    timezone_from_locale(Cldr.LanguageTag.t() | Cldr.Locale.locale_name()) ::
  String.t() | {:error, {module(), String.t()}}

    Returns the time zone from a language tag or locale name.

    Arguments

    • locale is any language tag returned by Bonfire.Web.Cldr.Locale.new/1 or a locale name in the list returned by Bonfire.Web.Cldr.known_locale_names/0

    Returns

    • A time zone ID as a string or

    • :error if no time zone can be determined

    Examples 

    iex> Bonfire.Web.Cldr.Locale.timezone_from_locale "en-US-u-tz-ausyd"
"Australia/Sydney"
