    get(key, default \\ nil, opts \\ [])

    View Source

    Get config value for a config key (optionally from a specific OTP app or Bonfire extension)

    These two calls have the same result (i.e. specifying a module as the first key will add the OTP app of that module as the first key): get([:bonfire_me, Bonfire.Me.Users]) get(Bonfire.Me.Users)

    Same with these two (i.e. not specifying a module or app as the first key will default to the main OTP app): get([:random_atom, :sub_key]) get([:bonfire, :random_atom, :sub_key])

    keys_tree(keys)

    View Source

    Constructs a key path for settings/config, which always starts with an app or extension name (which defaults to the main OTP app)

    iex> Bonfire.Me.Settings.keys_tree([:bonfire_me, Bonfire.Me.Users]) [:bonfire_me, Bonfire.Me.Users]

    iex> Bonfire.Me.Settings.keys_tree(Bonfire.Me.Users) [:bonfire_me, Bonfire.Me.Users]

    iex> Bonfire.Me.Settings.keys_tree(:bonfire_me) [:bonfire_me]

    iex> Bonfire.Me.Settings.keys_tree(:random_atom) [:bonfire, :random_atom]

    iex>Bonfire.Me.Settings.keys_tree([:random_atom, :sub_key]) [:bonfire, :random_atom, :sub_key]
